2. Descripción detallada de la actividad de aprendizaje
Con el desarrollo de esta actividad se espera que se alcance el siguiente
resultado de aprendizaje:
• Resultado de aprendizaje 2: Diseñar e implementar soluciones de
almacenamiento y procesamiento de grandes volúmenes de datos
utilizando herramientas como Hadoop, Spark y Kafka, para
desarrollar infraestructuras tecnológicas que soporten el análisis
eficiente de datos.
La actividad consiste en:
1. Investigación:
1
• Elaborar una tabla comparativa que resalte las diferencias clave
entre Hadoop y Spark en términos de arquitectura,
procesamiento, rendimiento y casos de uso
• Definir los conceptos de RDD. Propiedades de los RDDs:
Inmutabilidad, particionamiento. Tipos de Operaciones:
o Transformaciones: map, filter, flatMap, groupByKey,
reduceByKey.
o Acciones: collect, count, take, reduce.
• Concepto de DataFrame. Diferencias y ventajas sobre los RDDs.
• Crear un diagrama que ilustre la arquitectura de Kafka y
explicar los conceptos clave como topics, partitions, brokers,
producers y consumers.
2. Definición del problema y conjunto de datos: Cada estudiante
deberá seleccionar un problema que pueda ser resuelto mediante
el análisis de un conjunto de datos de gran volumen.
• El conjunto de datos puede ser propio o provenir de alguna
fuente pública (Kaggle, UCI Machine Learning Repository, datos
abiertos de Colombia, etc.).
3. Implementación en Spark: Desarrollar una aplicación
implementado Spark (utilizando Python) que realicen las
siguientes tareas:
• Procesamiento en batch:
o Cargar el conjunto de datos seleccionado desde la fuente
original.
o Realizar operaciones de limpieza, transformación y
análisis exploratorio de datos (EDA) utilizando RDDs o
DataFrames.
o Almacenar los resultados procesados.
• Procesamiento en tiempo real (Spark Streaming &
Kafka):
o Configurar un topic en Kafka para simular la llegada de
datos en tiempo real (usar un generador de datos).
o Implementar una aplicación Spark Streaming que
consuma datos del topic de Kafka.
2
o Realizar algún tipo de procesamiento o análisis sobre los
datos en tiempo real (contar eventos, calcular
estadísticas, etc.).
o Visualizar los resultados del procesamiento en tiempo
real.
Para el desarrollo de este componente practico deberá acceder al
siguiente documento:
• Anexo 3. Instructivo Spark Streaming y Kafka. Este
documento contiene el paso a paso de como instalar y
configurar Apache Streaming y Kafka para el posterior
procesamiento de datos.
4. Documentación y presentación:
• Foro de discusión: Compartir los avances del proyecto y
socializar el conjunto de datos seleccionado, problemas
encontrados y soluciones implementadas.
• Repositorio de código: Cada estudiante debe publicar el
código fuente del proyecto en un repositorio Git (GitHub,
GitLab, etc.) con una descripción clara de la solución y las
instrucciones para su ejecución.
• Video Explicativo: Cada estudiante debe crear un video de
máximo 5 minutos donde explique claramente el código
desarrollado.
• Presentación online: Cada estudiante deberá elaborar una
presentación que incluya:
o Introducción al problema y explicación del conjunto de
datos seleccionado.
o Arquitectura de la solución en Spark.
o Enlace del código publicado (GitHub, GitLab, etc)
o Enlace del video explicativo del código desarrollado.
o Capturas de Pantalla de la visualización de los resultados
obtenidos con sus respectivos análisis (Máximo 5
capturas).
